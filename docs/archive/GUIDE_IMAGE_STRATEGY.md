# Azur Menton Guide Image Strategy

## Current approach

The Menton Guide does not scrape Google Maps, does not hotlink Google Maps photos, and does not use unofficial Google Maps screenshots as content.

Current guide visuals use:
- project-owned apartment and location images already stored in `/public/images`
- project event illustrations where relevant
- editorial placeholders generated by the local UI for topics without owned imagery
- external Google Maps search links for navigation only

Place cards can show owned local images when available. If no suitable owned image exists, the UI renders a polished editorial placeholder based on the place or article theme.

## Google Maps / Places policy

Do not hotlink Google Maps image URLs. Do not scrape photos from Google Maps. Do not use screenshots from Google Maps as guide imagery.

A future official Google Places / Place Photos integration can be considered, but it must use Google Maps Platform properly and comply with attribution, caching, billing and privacy requirements.

A future implementation would require:
- Google Maps Platform API key
- billing enabled on the Google Cloud project
- attribution compliance for Google content
- request caching rules compliant with Google Maps Platform policies
- privacy and cookie review before deployment
- EEA terms review where applicable
- clear fallback behavior when Google content is unavailable

## Future implementation notes

The project includes a non-active `src/lib/googlePlaces.ts` skeleton for future typed integration work. It does not call Google APIs, does not require environment variables, and is not imported by production UI.
